# Pilates Site 최소 동작 체크리스트

## 1) 기본 로딩
- [ ] 브라우저에서 `index.html`이 정상적으로 열린다.
- [ ] 상단 헤더/히어로/각 섹션 스타일이 깨지지 않는다.

## 2) Program -> Truly 연동
- [ ] `Program` 카드의 `자세히` 버튼 클릭 시 `Truly` 섹션으로 스크롤된다.
- [ ] 선택한 프로그램의 제목/한줄소개/가격/권장빈도가 변경된다.

## 3) 아코디언 동작
- [ ] `효과/대상/주의사항/진행 방식` 버튼 클릭 시 패널이 열린다.
- [ ] 아이콘이 `＋`에서 `－`로 변경된다.
- [ ] 다시 클릭하면 패널이 닫히고 아이콘이 `＋`로 복귀한다.

## 4) 예약 조회
- [ ] 샘플 1: `R-1029` / `1234` 조회 성공.
- [ ] 샘플 2: `R-2031` / `7777` 조회 성공.
- [ ] 잘못된 값 입력 시 "일치하는 예약이 없습니다" 메시지가 보인다.

## 5) 변경/취소 접수
- [ ] `변경` 타입에서 날짜/시간 없이 제출하면 경고가 표시된다.
- [ ] 정상 제출 시 `REQ-...` 형식의 요청번호가 표시된다.
- [ ] 제출 후 폼이 초기화된다.

## 6) 부가 동작
- [ ] 우측 상단 `↑` 버튼 클릭 시 페이지 최상단으로 이동한다.
- [ ] `카카오 상담하기` 클릭 시 안내 alert가 표시된다.

## 7) 로컬 저장 확인(데모)
- [ ] 변경/취소 제출 후 개발자도구 `Application > Local Storage`에서 `pilates_requests` 키가 생성된다.
- [ ] 페이지 새로고침 후에도 `pilates_requests` 데이터가 유지된다.
